Edge Deployment

Overview
Explain strategies for deploying vision models to edge devices including model optimization, packaging, and runtime constraints.

Why This Topic Matters
Edge deployment brings vision intelligence closer to sensors, but it requires careful consideration of size, latency and device limitations.

Lecture Goals
- Define edge deployment scenarios and constraints
- Optimize models for size and speed
- Select appropriate runtime frameworks
- Package models for edge inference
- Monitor edge performance and reliability

Key Concepts
- model quantization
- pruning
- runtime optimization
- device constraints
- deployment packaging

Classroom Example / Case Study
- Discuss the trade-offs between accuracy and latency on edge hardware
- Explain quantization and its effect on model output
- Review edge runtimes such as ONNX Runtime, TensorRT, and TFLite

Practical Implementation Notes
- Test optimized models on representative hardware.
- Use benchmarks that match expected input sizes.
- Keep configuration and preprocessing identical to the training pipeline.

Tools & Libraries
- ONNX Runtime
- TensorFlow Lite
- TensorRT
- OpenVINO
- NVIDIA Jetson
