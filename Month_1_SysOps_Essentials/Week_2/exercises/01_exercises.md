 Week 2 Exercises: Linux, Bash, and System Automation

 Objectives
Apply the week’s concepts through practical, hands-on tasks that reinforce Linux command-line, Bash scripting, system administration, and automation skills.

 1. Linux Navigation & Inspection
- Use `ls`, `cd`, `pwd`, `find`, and `tree` to explore the filesystem.
- Use `cat`, `less`, `head`, `tail`, and `wc` to inspect file contents.
- Change file permissions and ownership with `chmod` and `chown`.

 2. Bash Scripting Basics
- Write a Bash script that takes a directory as an argument and lists all files larger than 10MB.
- Add error handling for missing or invalid arguments.
- Log the output to a file with a timestamp.

 3. Text Processing & Automation
- Use `grep`, `awk`, and `sed` to extract and summarize information from system logs (e.g., `/var/log/syslog` or `/var/log/auth.log`).
- Write a script that counts failed login attempts and outputs a summary report.

 4. System Administration Practice
- Add a new user, set a password, and add the user to a group.
- Start, stop, and check the status of a system service (e.g., `nginx`, `sshd`, or `cron`).
- Document each step and include command output.

 5. Automation & Scheduling
- Set up a cron job to run your log analysis script every hour.
- Verify that the job runs and logs output as expected.

 6. Reflection
- Write a short summary in your `README.md` about what you learned, challenges faced, and how you resolved them.
