 Week 2 Project: Linux & Automation System Report

 Objectives
- Demonstrate mastery of Linux fundamentals, Bash scripting, system administration, and shell automation.
- Build, document, and present a real-world automation or system management solution.
- Emphasize reproducibility, security, and operational best practices.

 Project Tasks
1. System Audit Script:
	- Write a Bash script that audits key aspects of a Linux system (users, disk usage, running services, open ports, etc.).
	- The script should log results to a timestamped file and handle errors gracefully.

2. Automation Workflow:
	- Create a second script or use cron/systemd to automate a routine task (e.g., log rotation, backup, user report, or service health check).
	- Document how to schedule and monitor the automation.

3. System Administration Scenario:
	- Demonstrate user and group management (add, modify, delete users/groups; set permissions).
	- Show how to install, start, and monitor a system service (e.g., nginx, sshd).

4. Documentation:
	- Provide a `README.md` with setup instructions, script usage, and troubleshooting tips.
	- Include an architecture diagram or flowchart of your automation workflow.
	- Summarize lessons learned and any challenges faced.

 Deliverables
- Bash scripts and any supporting files
- Sample log/output files
- `README.md` with documentation and diagrams
- Screenshots or terminal output as evidence

 Evaluation Criteria
| Item | Weight |
|------|--------|
| System audit and automation scripts | 30% |
| System administration scenario | 20% |
| Documentation and diagrams | 25% |
| Reproducibility, error handling, and best practices | 25% |
