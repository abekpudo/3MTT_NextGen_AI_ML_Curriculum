 Week 1 Exercises: Orientation, Environment, Git & GitHub Collaboration

 Objectives
Apply the week’s concepts through practical, hands-on tasks that reinforce orientation, environment setup, Git fundamentals, and GitHub collaboration.

 1. Orientation & Planning
- Read the program guide and list three key learning outcomes for this course.
- Create a Markdown file (`journal.md`) to record your goals and any questions you have about the curriculum.

 2. Environment Setup
- Install Python (3.8+), VS Code, and Git on your system.
- Create a virtual environment and activate it.
- Install at least one package (e.g., `requests`) and verify the installation.
- Document your setup steps in a `SETUP.md` file.

 3. Git Fundamentals
- Initialize a new Git repository in a sample project folder.
- Make three commits, each with a clear, descriptive message.
- Create a new branch (`feature/hello-world`), make a change, and merge it back to main.
- Use `git log` and `git diff` to inspect your commit history and changes.

 4. GitHub Collaboration
- Create a new repository on GitHub and push your local repo to it.
- Open an issue describing a feature or bug.
- Open a pull request from your feature branch, referencing the issue.
- Assign a reviewer (yourself or a peer) and respond to feedback.
- Set up a simple GitHub Project board or use Issues to track your work.

 5. Reflection
- Write a short summary in your `README.md` about what you learned, any challenges faced, and how you resolved them.
