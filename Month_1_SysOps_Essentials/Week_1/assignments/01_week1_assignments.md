 Week 1 Assignment: Foundations, Environment, Git & GitHub Collaboration

 Objectives
- Demonstrate understanding of program orientation, environment setup, Git fundamentals, and GitHub collaboration.
- Set up a reproducible development environment and document the process.
- Apply version control and collaborative workflows in a real project scenario.

 Assignment Tasks
1. Orientation & Planning:
	- Review the program guide and create a personal learning journal (Markdown file) outlining your goals and expectations for the course.
	- Summarize the curriculum flow and key resources in your own words.

2. Environment Setup:
	- Install Python, VS Code, Git, and any required CLI tools (e.g., Docker, AWS CLI).
	- Create a virtual environment and document the setup steps in a `SETUP.md` file.
	- Take a screenshot of your configured environment (terminal, editor, etc.) and include it in your submission.

3. Git Fundamentals:
	- Initialize a new Git repository for your sample project.
	- Make at least three meaningful commits, each with a clear message.
	- Create and switch to a feature branch, then merge it back to main using a pull request (if working solo, simulate this on GitHub).
	- Use `git log` and `git diff` to review your commit history and changes.

4. GitHub Collaboration:
	- Push your repository to GitHub (public or private).
	- Open a pull request with a descriptive title and summary, referencing an issue (create one if needed).
	- Assign a reviewer (can be a peer or yourself for practice) and respond to review comments.
	- Set up a simple GitHub Project board or use Issues to track tasks.

5. Documentation & Reflection:
	- Write a concise README that explains your setup, workflow, and any challenges faced.
	- Document any troubleshooting steps or lessons learned.

 Evaluation Criteria
| Item | Weight |
|------|--------|
| Personal learning journal & orientation summary | 10% |
| Environment setup & documentation | 20% |
| Git repository with meaningful commits & branching | 20% |
| GitHub collaboration (PR, issues, project board) | 25% |
| README & troubleshooting/reflection | 25% |
