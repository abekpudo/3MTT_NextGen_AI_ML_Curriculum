 Month 1: SysOps Essentials

Duration: 4 Weeks | Theme: Infrastructure & Professional Development | Credit Hours: Foundation

---

 🎯 Month Objectives

By the end of this month, students will:
- ✅ Master Git workflow and GitHub collaboration
- ✅ Become comfortable with Linux/Bash command line
- ✅ Understand cloud computing fundamentals (AWS)
- ✅ Write professional technical documentation
- ✅ Set up professional development environment
- ✅ Understand DevOps principles and CI/CD basics

 📚 Weekly Breakdown

 Week 1: Foundations & Program Orientation 🚀

Learning Objectives:
- Understand the 3MTT NextGen program structure
- Set up development environment
- Learn Git fundamentals
- Create GitHub profile and first repository

Topics:
1. Cohort Orientation (1 day)
   - Program overview and expectations
   - Curriculum structure and timeline
   - Assessment and grading
   - Community and support systems

2. Development Environment Setup (1-2 days)
   - Python installation (3.9 or higher)
   - IDE selection (VS Code recommended)
   - Package management (pip, conda)
   - Jupyter Notebook setup

3. Introduction to Git (1-2 days)
   - What is version control and why it matters
   - Git basics: init, add, commit, log
   - Understanding git workflow
   - .gitignore files and best practices

4. GitHub & Collaboration (1-2 days)
   - GitHub account creation
   - Repository creation and cloning
   - Push, pull, and fork workflows
   - Pull requests and code review
   - Issues and project management

Hands-On Activities:
- [ ] Clone the curriculum repository
- [ ] Create personal GitHub profile with professional bio
- [ ] Make first commit and push to repository
- [ ] Create a practice repository with multiple commits
- [ ] Practice forking and creating pull requests

Deliverables:
- [ ] GitHub profile completed (profile README, photo)
- [ ] First repository created with proper structure
- [ ] At least 5 commits with meaningful messages
- [ ] README.md explaining your repository

Resources:

 Week 2: Linux, Bash & Command Line Mastery 💻

Learning Objectives:
- Navigate Linux filesystem confidently
- Use Bash for productive development
- Master essential command line tools
- Understand Unix philosophy and shell scripting basics

Topics:
1. Linux Fundamentals (1 day)
   - What is Linux and distributions
   - Installation and setup
   - File system hierarchy
   - File permissions and ownership

2. Bash Basics (2 days)
   - Shell basics and bash profile
   - Essential commands (ls, cd, cp, mv, rm, mkdir)
   - File operations and manipulation
   - Wildcards and special characters

3. Advanced Bash (1 day)
   - Pipes and redirection
   - grep, sed, awk fundamentals
   - Shell scripting basics
   - Automating common tasks

4. Text Processing (1 day)
   - Working with text files
   - grep, find, and locate
   - Stream editing with sed
   - Pattern matching with awk

Practical Scenarios:
- Navigate to a specific directory and list all files modified in the last week
- Create directory structure automatically using Bash scripts
- Parse CSV data using command-line tools
- Automate file backup using shell scripts
- Manage file permissions for a project structure

Hands-On Activities:
- [ ] Create 10+ directories with various structures
- [ ] Write a Bash script that performs useful automation
- [ ] Use pipes to chain multiple commands
- [ ] Extract and transform data using command-line tools
- [ ] Set up git hooks using Bash scripts

Deliverables:
- [ ] Bash script repository with 3-5 useful scripts
- [ ] Documentation of each script's purpose
- [ ] Terminal commands cheat sheet created
- [ ] Proof of successful script execution

Resources:
- [Bash Guide for Beginners](http://tldp.org/LDP/Bash-Beginners-Guide/html/)
- [Linux Journey](https://linuxjourney.com/) - Interactive Linux learning
- [Ryan's Tutorials - Linux](https://ryanstutorials.net/linuxtutorial/)
- [Bash Scripting Tutorial](https://www.shellscript.sh/)

 Week 3: AWS Cloud Fundamentals ☁️

Learning Objectives:
- Understand cloud computing concepts
- Set up AWS account and navigate console
- Use EC2, S3, and IAM services
- Understand security best practices
- Deploy and manage basic cloud resources

Topics:
1. Cloud Computing Basics (1 day)
   - What is cloud computing
   - IaaS, PaaS, SaaS explained
   - AWS overview and services
   - Regions and Availability Zones

2. AWS Account & Security (1 day)
   - AWS account creation and setup
   - IAM users and roles
   - Access keys and security credentials
   - AWS security best practices
   - Cost monitoring and billing

3. Compute Services: EC2 (1 day)
   - EC2 instances and instance types
   - Security groups and key pairs
   - Elastic IPs and network configuration
   - Launching and managing instances
   - SSH access and remote management

4. Storage Services: S3 & EBS (1 day)
   - S3 buckets and objects
   - S3 permissions and access control
   - EBS volumes and snapshots
   - Data transfer and lifecycle policies

AWS Services to Master:
| Service | Purpose | Use Case |
|---------|---------|----------|
| EC2 | Compute/Virtual Machines | Running ML training jobs |
| S3 | Object Storage | Storing datasets and models |
| RDS | Managed Databases | Data storage for applications |
| Lambda | Serverless Computing | Event-driven model inference |
| SageMaker | ML Platform | End-to-end ML workflows |
| CloudWatch | Monitoring | Track application metrics |

Hands-On Activities:
- [ ] Create AWS account with billing alerts
- [ ] Create IAM users with appropriate permissions
- [ ] Launch EC2 instance and SSH into it
- [ ] Install Python and run a script on EC2
- [ ] Upload and download files from S3
- [ ] Create and manage S3 buckets
- [ ] Configure security groups properly
- [ ] Monitor EC2 instance metrics

Project: Deploy Simple Web App
- Launch EC2 instance
- Install dependencies (Python, Flask)
- Deploy simple Flask application
- Access application from browser
- Document the process

Deliverables:
- [ ] AWS account set up with best practices
- [ ] Documented EC2 instance setup process
- [ ] S3 bucket with organized folder structure
- [ ] Security configuration documentation
- [ ] Cost estimation for sample workload

Resources:
- [AWS Free Tier Documentation](https://aws.amazon.com/free/)
- [AWS Skill Builder](https://skillbuilder.aws.com/) - Free training modules
- [Linux Academy / A Cloud Guru - AWS Essentials](https://www.pluralsight.com/paths/aws-fundamentals)
- [AWS Architecture Center](https://aws.amazon.com/architecture/)

 Week 4: Professional Documentation & DevOps Intro 📝

Learning Objectives:
- Write professional technical documentation
- Master Markdown formatting
- Understand documentation best practices
- Learn DevOps principles and CI/CD basics
- Set up project documentation structure

Topics:
1. Technical Writing (1 day)
   - Documentation types and purposes
   - Audience analysis
   - Clear and concise writing
   - Using examples and diagrams effectively

2. Markdown Mastery (1 day)
   - Markdown syntax and formatting
   - Creating structured documents
   - Code blocks and syntax highlighting
   - Tables, lists, and formatting tricks
   - Markdown tools and editors

3. Project Documentation (1 day)
   - README files - structure and content
   - API documentation
   - Setup and installation guides
   - Architecture diagrams
   - Contributing guidelines

4. Introduction to DevOps (1 day)
   - DevOps culture and mindset
   - CI/CD concepts and benefits
   - Version control and deployment
   - Continuous integration basics
   - GitHub Actions introduction

Documentation Types:

| Document Type | Purpose | Audience |
|---------------|---------|----------|
| README.md | Project overview | Everyone |
| INSTALL.md | Setup instructions | Developers |
| API.md | API documentation | Developers |
| CONTRIBUTING.md | How to contribute | Contributors |
| ARCHITECTURE.md | System design | Technical team |
| USER_GUIDE.md | How to use | End users |

Hands-On Activities:
- [ ] Write professional README for a sample project
- [ ] Create API documentation for mock service
- [ ] Write setup and installation guide
- [ ] Create architecture diagram using Markdown
- [ ] Write contributing guidelines
- [ ] Create GitHub Actions workflow
- [ ] Set up automatic testing on push

Documentation Project:
Create comprehensive documentation for Month 1 project including:
- [ ] README with project overview
- [ ] Installation and setup guide
- [ ] Usage examples and tutorials
- [ ] Architecture and design decisions
- [ ] Troubleshooting guide
- [ ] Contributing guidelines

DevOps Concepts:
- Continuous Integration (CI): Automatically test code
- Continuous Deployment (CD): Automatically deploy code
- Infrastructure as Code: Manage infrastructure with version control
- Monitoring and Logging: Track application performance
- Containerization: Docker for consistent environments

GitHub Actions Practice:
- [ ] Create workflow for running tests
- [ ] Set up automatic code formatting
- [ ] Configure deployment on push
- [ ] Create notification workflows

Deliverables:
- [ ] Complete project documentation (README, guides, API docs)
- [ ] GitHub Actions workflow file
- [ ] Architecture diagram
- [ ] Contributing guidelines
- [ ] Monthly reflection and documentation of learning

Resources:
