Scaling

Overview
Explain how to scale data and model systems horizontally and vertically while controlling cost, latency, and reliability.

Why This Topic Matters
Scaling is a key concern for production systems, as higher loads expose bottlenecks and require architecture adjustments.

Lecture Goals
- Differentiate vertical and horizontal scaling
- Use autoscaling based on load and resource metrics
- Optimize data partitioning for distributed jobs
- Balance latency and throughput requirements
- Plan for capacity and cost management

Key Concepts
- horizontal scaling
- vertical scaling
- autoscaling
- partitioning
- resource optimization

Classroom Example / Case Study
- Describe how adding nodes changes distributed processing capacity
- Explain when to scale compute versus storage separately
- Use metrics to make scaling decisions rather than guesswork

Practical Implementation Notes
- Scale only when needed and after identifying bottlenecks.
- Maintain cost visibility as systems grow.
- Test scaling behaviour with realistic load patterns.

- autoscaling groups
- Spark cluster scaling
- cloud monitoring
- load testing
