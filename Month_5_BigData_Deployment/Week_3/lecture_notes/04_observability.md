Observability

Overview
Cover observability for deployed data and ML systems, including telemetry, logs, traces, and alerting.

Why This Topic Matters
Observability gives teams the insight needed to detect issues early, understand system behaviour, and keep services reliable.

Lecture Goals
- Define observability signals: metrics, logs, traces
- Instrument services for telemetry
- Build dashboards and alerts
- Correlate failures across components
- Use observability for performance tuning

Key Concepts
- metrics
- logs
- traces
- instrumentation
- alerting

Classroom Example / Case Study
- Explain how to collect and aggregate metrics from the model serving stack
- Describe trace propagation across API requests and data pipelines
- Show how to use dashboards to diagnose slow or failing workflows

Practical Implementation Notes
- Instrument systems from the start, not after a failure.
- Use business-level metrics combined with system metrics.
- Make alerts actionable with clear remediation instructions.

Tools & Libraries
- Prometheus
- Grafana
- OpenTelemetry
- Cloud monitoring
- Alertmanager
