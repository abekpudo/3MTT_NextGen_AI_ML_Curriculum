 📚 How to Use This Curriculum

 For Each Week:

1. Read the Overview
   - Start with `Week_X/README.md` in each month folder
   - Understand weekly objectives
   - Review the topics to be covered

2. Study Lecture Notes
   - Read through lecture notes in `lecture_notes/` folder
   - Take notes, highlight key concepts
   - Identify areas that need deeper understanding

3. Complete Exercises
   - Work through `exercises/` folder
   - Practice concepts with hands-on code
   - Don't skip exercises! They're crucial for learning

4. Do Assignments
   - Complete `assignments/` folder
   - These are graded components
   - Submit via GitHub (instructions provided)

5. Use Cheatsheets
   - Reference `cheatsheets/` for quick lookups
   - Print and keep nearby during coding
   - Create your own cheatsheets too!

6. Review & Reflect
   - Summarize what you learned
   - Write in your learning journal
   - Identify remaining questions

 🤝 Getting Help

 Resources in Order of Recommendation

1. Documentation & Lecture Notes
   - First place to look for answers
   - Usually has examples and explanations

2. Cheatsheets
   - Quick reference for syntax and concepts
   - Organized by topic

3. Google & Stack Overflow
   - Vast collection of solutions
   - Often has exactly what you need
   - Learn to search effectively

4. GitHub Issues & Discussions
   - Ask questions in course repo
   - Get help from instructors and peers
   - Learn from others' questions

5. Discord Channel
   - Real-time help and discussion
   - Study groups forming
   - Share resources and wins

6. Office Hours
   - Schedule with instructors
   - Go prepared with specific questions
   - 30-minute sessions available

7. Peer Study Groups
   - Find study buddies
   - Work through challenges together
   - Teach each other concepts

 Asking Effective Questions

Good question format:
```
Issue: [Clear description of problem]

What I tried: [What approaches have you attempted?]

Expected vs Actual: [What should happen vs what happens]

Environment: [OS, Python version, relevant packages]

Minimal code example:
[Short, runnable code that demonstrates issue]
```

 🎓 Learning Tips

 Best Practices for Success

1. Code Every Day - Consistency beats intensity
2. Read Documentation - Before asking for help
3. Experiment Actively - Don't just read, try variations
4. Teach Others - Best way to solidify learning
5. Track Your Progress - Celebrate small wins
6. Take Breaks - Rest is part of learning
7. Review Regularly - Spaced repetition works
8. Ask Questions - Curiosity drives deeper understanding
9. Build Things - Projects reinforce concepts
10. Have Fun - Enjoy the learning journey!

 Study Tips by Learning Style

Visual Learners:
- Use Jupyter Notebooks with lots of visualizations
- Create diagrams and flowcharts
- Watch video tutorials
- Use colorful note-taking

Auditory Learners:
- Explain concepts out loud
- Join study groups and discussions
- Listen to lectures multiple times
- Teach classmates

Kinesthetic Learners:
- Hands-on coding immediately
- Build projects early and often
- Practice with real datasets
- Interactive exercises first

 Before Starting

1. Set Clear Goals
   - "I want to build ML models" is vague
   - "I'll master scikit-learn and get 90% accuracy" is better
   - Write your goals and review weekly

2. Create Accountability
   - Study buddy or group
   - Share progress on social media
   - Check in with instructors

3. Prepare Your Environment
   - Quiet space for concentrated work
   - Multiple monitors (if possible)

2. Build a Portfolio
